# 需求文档

## 1. 应用概述

**应用名称**：自助点餐系统

**应用描述**：用于线下场景（如文化祭、咖啡摊等）的自助点餐与订单管理系统。用户通过网页自助浏览菜单、下单并完成支付引导，店员通过管理员后台确认订单、管理菜单、查看出餐状态及统计数据。系统采用轻量实现方案，不依赖支付接口，以「扫码支付+人工确认」为核心，适用于临时活动场景。

## 2. 用户与使用场景

**目标用户**：
- 用户端：线下活动的顾客
- 管理员端：店员、活动组织者

**核心使用场景**：
- 用户场景：顾客在活动现场通过网页浏览菜单、选择商品、提交订单、完成支付引导、出示订单给店员确认
- 管理员场景：店员通过后台确认支付、管理订单状态、编辑菜单、查看出餐屏、统计销售数据、配置支付方式

## 3. 页面结构与功能说明

### 3.1 页面结构

```
自助点餐系统
├── 用户端
│   ├── 首页（菜单浏览）
│   ├── 购物车页面
│   ├── 订单确认页面
│   ├── 支付引导页面
│   └── 订单提交成功页面
└── 管理员端
    ├── 登录页面
    ├── 订单管理页面（打工模块）
    ├── 菜单编辑页面（编辑模块）
    ├── 出餐展示页面（展示模块）
    ├── 统计页面（计算模块）
    └── 支付配置页面
```

### 3.2 用户端功能说明

#### 3.2.1 首页（菜单浏览）

**功能**：
- 显示一级分类与商品列表
- 商品信息包含：名称、价格、图片、描述
- 点击商品可加入购物车
- 显示购物车图标及商品数量
- 特殊项「呼叫店员」：点击提示「请呼叫身边店员」，长按3秒+连点5次进入管理员登录入口

#### 3.2.2 购物车页面

**功能**：
- 显示已选商品列表
- 每个商品显示：名称、单价、数量、小计
- 支持修改数量（+/-按钮）
- 支持删除单项商品
- 支持清空购物车
- 显示订单总价
- 点击「确认订单」进入订单确认页面

#### 3.2.3 订单确认页面

**功能**：
- 显示订单商品明细与总价
- 选择支付方式（微信/支付宝/PayPay/现金）
- 点击「提交订单」进入支付引导页面

#### 3.2.4 支付引导页面

**功能**：
- 显示选择的支付方式对应的二维码（微信/支付宝/PayPay）或现金支付提示
- 显示订单金额
- 用户自行完成支付后，点击「我已支付」按钮
- 订单状态变更为「待确认支付」
- 进入订单提交成功页面

#### 3.2.5 订单提交成功页面

**功能**：
- 显示订单编号
- 提示用户出示订单给店员确认
- 点击「确认」或60秒无操作后返回首页

### 3.3 管理员端功能说明

#### 3.3.1 登录页面

**功能**：
- 输入密码登录
- 密码由后端校验
- 登录成功后进入订单管理页面

#### 3.3.2 订单管理页面（打工模块）

**功能**：
- 显示订单列表，包含：订单编号、商品明细、总价、当前状态、创建时间
- 支持按状态筛选订单（已创建、待确认支付、正在制作、待领取、已完成、已取消）
- 点击订单查看详情
- 修改订单状态：待确认支付 → 正在制作 → 待领取 → 已完成
- 取消订单（状态变更为「已取消」）
- 删除订单（状态变更为「已删除」，订单不再显示）
- 完成订单时需二次确认

#### 3.3.3 菜单编辑页面（编辑模块）

**功能**：
- 分类管理：新增、编辑、删除分类
- 商品管理：新增、编辑、删除商品
- 商品信息包含：名称、价格、图片、描述、所属分类
- 支持上传商品图片

#### 3.3.4 出餐展示页面（展示模块）

**功能**：
- 分区显示订单：正在制作、待领取
- 每个订单显示订单编号
- 已完成订单不显示
- 页面每3秒自动刷新

#### 3.3.5 统计页面（计算模块）

**功能**：
- 显示商品销量（按商品统计已完成订单中的数量）
- 显示商品销售额（按商品统计已完成订单中的金额）
- 显示总营业额（所有已完成订单的总金额）
- 数据实时更新

#### 3.3.6 支付配置页面

**功能**：
- 二维码管理：上传微信、支付宝、PayPay的静态二维码图片，支持替换图片，实时生效
- 支付方式控制：启用/禁用微信、支付宝、PayPay、现金支付方式
- 未配置或禁用的支付方式在用户端不显示

## 4. 视觉设计规范

### 4.1 设计基准

- 以横版PC为主设计基准
- 字号醒目，适合触屏阅读
- 参考文件：type.txt中的美术风格

### 4.2 色彩系统

**背景色**：
- 主背景：深暗暖褐色系（#171411及相近色）
- 卡片背景：毛玻璃材质，半透明暖褐色

**高亮色**：
- 主高亮：暖金色（#F4B46A）
- 次高亮：琥珀色（#D99B65）
- 用于按钮、标题、重要信息强调

**文字色**：
- 主文字：暖白色/米色系
- 次要文字：低饱和度暖灰色

### 4.3 材质与肌理

**卡片材质**：
- 毛玻璃效果（半透明、模糊背景）
- 应用于商品卡片、订单卡片、功能模块卡片

**背景肌理**：
- 有机噪点肌理叠加
- 环境流光背景（柔和渐变光晕）

### 4.4 字体规范

**标题字体**：
- 使用Noto Serif SC（思源宋体）
- 应用于页面标题、模块标题、重要信息标题

**正文字体**：
- 使用无衬线字体（如思源黑体、苹方等）
- 应用于商品描述、订单详情、表单文字

### 4.5 动效规范

**呼吸感微动效**：
- 应用于按钮、卡片悬停状态
- 轻微缩放或透明度变化
- 动画时长：0.3-0.6秒

**流光效果**：
- 背景环境光缓慢流动
- 营造温暖氛围感

### 4.6 应用范围

**用户端**：
- 首页菜单浏览、购物车、订单确认、支付引导、订单成功页面均应用上述视觉规范

**管理员端**：
- 登录页面、订单管理、菜单编辑、出餐展示、统计页面、支付配置页面均应用上述视觉规范

**注意事项**：
- 不引入type.txt中的功能元素（如AI SCORE、推荐分等）
- 仅更新视觉样式，不改变页面结构与功能逻辑

## 5. 业务规则与逻辑

### 5.1 订单状态流转规则

**状态定义**：
- 已创建（未支付）：订单已提交，用户未点击「我已支付」
- 待确认支付：用户点击「我已支付」，等待店员确认
- 正在制作：店员确认支付后，订单进入制作状态
- 待领取：商品制作完成，等待用户领取
- 已完成：用户已领取商品，订单完成
- 已取消：订单被取消，参与数据记录但不参与统计
- 已删除：订单被删除，仅隐藏不显示，不参与前端展示

**流转规则**：
- 状态必须按顺序流转：已创建 → 待确认支付 → 正在制作 → 待领取 → 已完成
- 不允许跳跃修改状态
- 仅「已完成」状态的订单参与统计
- 「已删除」状态的订单不在前端显示

### 5.2 支付系统规则

- 系统不对接支付接口，仅提供二维码与金额展示
- 用户自行完成支付后点击「我已支付」
- 店员通过管理员后台人工确认支付
- 支持的支付方式：微信、支付宝、PayPay（扫码）、现金
- 支付方式由管理员在支付配置页面启用/禁用
- 二维码为静态二维码，不生成动态金额二维码

### 5.3 页面重置规则

- 用户端在订单提交成功页面点击「确认」或60秒无操作后，自动返回首页
- 购物车数据清空

### 5.4 数据同步规则

- 前端每3秒轮询后端获取最新订单状态
- 出餐展示页面每3秒自动刷新
- 统计数据实时更新

### 5.5 管理员权限规则

- 管理员登录需输入密码，密码由后端校验
- 登录后可访问所有管理员功能模块

## 6. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 购物车为空时点击「确认订单」 | 提示「购物车为空，请先添加商品」 |
| 未选择支付方式时点击「提交订单」 | 提示「请选择支付方式」 |
| 管理员密码输入错误 | 提示「密码错误，请重新输入」 |
| 删除分类时该分类下有商品 | 提示「该分类下有商品，无法删除」 |
| 修改订单状态时状态不符合流转规则 | 提示「状态流转错误，请按顺序操作」 |
| 网络异常导致数据未同步 | 提示「网络异常，请稍后重试」 |
| 上传图片格式不支持 | 提示「仅支持JPG、PNG格式图片」 |
| 支付方式未配置二维码 | 该支付方式在用户端不显示 |

## 7. 验收标准

1. 用户在首页浏览菜单，选择商品加入购物车
2. 用户进入购物车页面，确认商品与总价，点击「确认订单」
3. 用户在订单确认页面选择支付方式，点击「提交订单」
4. 用户在支付引导页面查看二维码与金额，完成支付后点击「我已支付」
5. 用户在订单提交成功页面查看订单编号，出示给店员
6. 店员在管理员后台查看订单，确认支付后将订单状态修改为「正在制作」
7. 店员制作完成后将订单状态修改为「待领取」
8. 用户领取商品后，店员将订单状态修改为「已完成」
